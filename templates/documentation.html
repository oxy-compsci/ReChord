1. Common Errors

Here are some common issue that you would like to know for making a perfect search:

1.1 Snippet Search:

  • Note that Snippet search only cares about pname of the notes. Not other attributes.
  • The snippet you used to search is required to be a full .xml or .mei because the snippet is first parsed to a tree and then search against.
  • Here is a simple acceptable snippet sample.

1.2 Terms Search:

    • At the moment, ReChord do not support "Hairpin" and "Piano Fingering" term search.
    • Terms search currently allow following categories:
      • Expressive Terms
      • Articulation
      • Dynamic Marking
      • Tempo Marking
      • Pedal Marking
  • Not only do we allow exact term search, ReChord also has a built in dictionary to search for the acronyms/short-hand writings and vise versa.

1.3 Upload Your Own Database:

  • If you are not uploading files to do search in, ReChord will search within MEI 3.0.0. Complete examples.
  • Also, the MEI file you upload shall also be complete and parsable.
  • You can upload multiple files at the same time, just be mindful that the files need to have .mei or .xml extension.
  • The files you are upload are temporary and can only support one search on the app page.
    • If you have a database and would like to do frequent search within, please:
      1. clone your own branch of our Git repository
      2. replace database/MEI_complete_example with your own folder of files
      3. if you no longer use the original folder name:
        • replace "MEI_Complete_examples" in my_form_post() function in ReChord_frontend.py with your own folder name.

1.4 If there are any other problem regarding this app:

  • Since this project only requires Python and xml knowledge, feel free to ask your programming friends technical question and let them looking into our code. It should be easily understandable and fixable. (and let them do a Pull Request please)
  • Feel free to report Issue on our git page.

2. Deploy the app through Heroku

This project is designed for users to deploy their own ReChord version through Heroku, a web app hosting cloud platform. This section will focus on how to deploy this GitHub repo on heroku. If you are a pro, just skip this session.

Step 1: Sign up and Log in

We believe you! You got this.

Step 2: Create a new app


Then on the managing projects page, add a new project. Heroku will prompt you to give a name for it and then you would enter the dash board.

Step 3: In Dashboard...

Step 3.1: Connect to GitHub

Then in the Deploy tab in the Dashboard, Click on GitHub for Deployment method.
and then connect to your GitHub account.

Step 3.2: Connect to your ReChord Git Branch

Step 3.3: One-time deployment

After connect to the branch, scroll down to find manually deploy session. Click on Deploy Branch. Then you are ready to go! Be mindful that this manual deployment only run with the version of code in the branch. This means that if you push to the branch again, the app will not deploy the new code. If you want to do so, you can click on Enable Automatic Deploys instead.
Now you can go to the setting and find the domain to check the app! Happy ReChording!
